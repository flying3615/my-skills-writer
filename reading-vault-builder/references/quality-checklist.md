# Reading Vault Quality Checklist

交付前逐项检查。任何一项明显不满足，都先修正再交付。

## 1. 来源与结构

- [ ] 已明确本次整理的是哪一本书、哪一个版本或哪一组材料
- [ ] 已建立章节或结构映射，而不是直接开始写笔记
- [ ] 如果材料不完整，已显式标注缺失部分
- [ ] 没有仅凭文件名猜测章节内容

## 2. 目录与覆盖

- [ ] 已创建 `ReadingVault/`
- [ ] `00-Overview/` 有全书总览页
- [ ] `01-Chapters/` 对每个主要章节都有笔记
- [ ] `02-Themes/` 只包含真正跨章节的主题
- [ ] `03-Concepts/` 只包含值得反复引用的概念/人物/框架
- [ ] `04-Quotes/` 的引文可追溯
- [ ] `05-Checks/` 只做轻量理解检查，不膨胀成题库

## 3. 章节笔记质量

- [ ] 每章都有摘要
- [ ] 每章都有核心观点、关键事件或关键推进
- [ ] 每章都有关键概念、人物或场景
- [ ] 每章都有与前后章节的联系
- [ ] 每章都有 `3-5` 个理解检查题
- [ ] 每章都有 `## Related Notes`

## 4. 主题与概念质量

- [ ] 主题页明确列出来源章节
- [ ] 主题页不是脱离章节上下文的空泛总结
- [ ] 概念页注明首次出现或主要出现章节
- [ ] 概念页解释了“为什么重要”，而不仅是复制定义

## 5. 引文与可追溯性

- [ ] 直接引文尽量附了页码或章节锚点
- [ ] 页码不稳定时，已明确写出按章节锚点定位
- [ ] 综合归纳已说明来自哪些章节
- [ ] 书外补充已标记为“补充说明”
- [ ] 没有伪造引文、页码或作者观点

## 6. 语言与表达

- [ ] 默认使用中文输出
- [ ] 原文术语、专有名词和关键引文被保留
- [ ] 首次出现的重要术语使用 `中文（Original Term）` 形式
- [ ] 区分“原书观点”和“助手解释”

## 7. 链接与导航

- [ ] `00-Overview/` 链接到章节页和主题页
- [ ] 章节页能链接到主题页或概念页
- [ ] 主题页反链到章节页
- [ ] 概念页能回到章节页或总览页
- [ ] 引文页能回到章节页

## 8. 结果是否符合目标

- [ ] 最终结果看起来像“可长期使用的读书笔记库”
- [ ] 没有漂移成聊天式陪读脚本
- [ ] 没有漂移成代码工程分析或软件 onboarding 文档
- [ ] 重点仍然是“章节笔记为主，主题索引为辅”
